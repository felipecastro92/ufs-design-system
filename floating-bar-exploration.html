Floating Utility Bar

Four structural approaches for a persistent bottom bar combining website search and LLM access. Each canvas is interactive — click inputs to focus, use toggles and FABs to explore states. All bars use DS tokens exclusively.

A — Centered Floating Pill

Both modes coexist side by side in a single pill, separated by a divider. The user's eye is always drawn to both affordances simultaneously. Clicking either side focuses it and activates the submit button for that mode. Minimal footprint, strong visual hierarchy.

The pill's white background is self-contained — it reads cleanly on any page background. No variant needed.

B — Single Input with Mode Toggle

One focused input bar with an inline mode selector on the left. The user deliberately chooses their intent before typing. Less spatial footprint than A — the two modes don't compete visually. Toggle the Search / Ask AI buttons below to see the mode transition.

Interactive — toggle the mode

Mode switch animates smoothly. The bar becomes narrower than A — better for pages where horizontal space matters.

C — Expandable FABs

Two floating action buttons anchored to the bottom-right corner. Tapping a FAB opens a compact input panel directly above it. The rest of the page stays fully visible and unobstructed when idle. Best for pages with dense content that shouldn't be covered.

D — Full-width Dock

A persistent bar pinned to the bottom edge spanning the full viewport width. Both modes are always visible and equally prominent — no toggle needed. The strongest in terms of discoverability. Two colour treatments below: dark (brand) and light (neutral).
