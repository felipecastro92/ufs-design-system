Overview

Header & Navigation System

Two parallel stacks — one for desktop (≥ 769 px) and one for mobile (≤ 768 px). Color themes via BEM modifier classes. Auth state and webshop mode control which action elements render.

🖥️
Desktop stack
.ufs-util + .ufs-nav. Utility bar sits above; nav is sticky. Both take the same color modifier.
📱
Mobile stack
.ufs-mnav (sticky bar) + .ufs-drawer (slide-in) + .ufs-msearch-overlay (full-screen search).
🎨
Color themes
--light = white bg, dark text. --glass = semi-transparent dark over food imagery with backdrop-blur.
💡 All colour values reference CSS custom properties from tokens.css, which ships alongside this document. Megamenus and search panels are siblings of <nav> (not children) — see the sections below for the correct wrapper pattern.

Journey Variants

Minimal Header — Landing & Registration

One shared header for both acquisition (Landing) and signup (Registration) journeys — no new component, just .ufs-nav reduced to a logo and a single text-link action. Utility bar, nav links/megamenu, search, favourites/cart, and any CTA button are all removed.

💡 Why one variant instead of two: both journeys reduce the header for the same reason — keep the visitor's attention on a single on-page action — and neither needs a header-level CTA anymore. Landing's CTA now lives in the sticky bottom bar below; Registration's CTA is the form's own submit button. The logo stays linked to home in both cases — low-stakes for Landing's top-of-funnel traffic, and simple/consistent to maintain as one component rather than branching logo behaviour per journey.

Sticky Bottom CTA

Landing's replacement for the header CTA — a full-width bar pinned to the bottom of the viewport, frosted glass over page content, reveal-on-scroll. Source: Figma — Desktop / Figma — Mobile.

💡 Reveal timing: hidden on load, appears once the hero's own CTA button scrolls out of view (via IntersectionObserver in the live prototype) — never shown at the same time as the hero CTA, so there's exactly one conversion ask on screen at any moment.

Checkout Header

A derivative of the Registration Form Header — same unlinked-logo and no-competing-CTA rules apply — adapted for an in-progress transaction rather than an in-progress signup.

💡 What changes vs. Registration: the leave-guard link becomes "Back to cart" instead of "Sign In" (the user is already past that step). The secure badge is always on, not optional — checkout is the exact moment price-anxiety peaks. Card-network logos are intentionally left out of the header (trademark/licensing overhead for a marginal trust gain — the lock badge carries most of that signal on its own). The progress indicator uses named steps on desktop, since "what's left" matters more here than in signup.

RTL Adaptation

Set dir="rtl" on <html>. Flexbox and Grid reverse automatically. The overrides below fix physical CSS properties that don't auto-mirror.

⚠️ Arabic typography: Use font-family: 'Cairo', sans-serif under [dir="rtl"]. Remove all letter-spacing (Arabic cursive breaks with tracking). Set line-height: ~1.75. Never apply synthetic italics or text-transform: uppercase to Arabic text.
Physical property overrides
CSS
/* Arabic font + reset letter-spacing */
[dir="rtl"] { font-family: 'Cairo', sans-serif; }
[dir="rtl"] * { letter-spacing: 0 !important; }

/* Logo: margin-right → margin-left */
[dir="rtl"] .ufs-nav__logo { margin-right: 0; margin-left: var(--space-7); }

/* Icon buttons: sibling margin flips */
[dir="rtl"] .ufs-nav__icon-btn + .ufs-nav__icon-btn { margin-left: 0; margin-right: var(--space-3); }
[dir="rtl"] .ufs-nav__actions .btn              { margin-left: 0; margin-right: var(--space-3); }

/* Badge + account-dot: flip to leading corner */
[dir="rtl"] .ufs-nav__badge       { right: auto; left: var(--space-1); }
[dir="rtl"] .ufs-nav__account-dot { right: auto; left: 6px; }

/* Language dropdown: anchor left */
[dir="rtl"] .ufs-lang-drop { right: auto; left: 0; }

/* Megamenu link arrows: mirror */
[dir="rtl"] .mm-link-arr { transform: scaleX(-1); }

/* Vertical chevrons: keep rotation (not directional flip) */
[dir="rtl"] .ufs-nav__link-chev                          { transform: rotate(90deg); }
[dir="rtl"] .ufs-nav__link.is-open .ufs-nav__link-chev     { transform: rotate(-90deg); }

/* Sign-in panel: enter from left in RTL */
[dir="rtl"] .signin-panel        { right: auto; left: 0; transform: translateX(-100%); }
[dir="rtl"] .signin-panel.is-open { transform: translateX(0); box-shadow: 8px 0 40px rgba(0,0,0,0.18); }

/* Drawer: slide from right in RTL */
[dir="rtl"] .ufs-drawer        { left: auto; right: 0; transform: translateX(100%); }
[dir="rtl"] .ufs-drawer.is-open { transform: translateX(0); }

/* Drawer chevrons */
[dir="rtl"] .ufs-drawer-acc-chev                              { transform: scaleX(-1); }
[dir="rtl"] .ufs-drawer-acc-trigger.is-open .ufs-drawer-acc-chev { transform: scaleX(-1) rotate(-90deg); }

/* Mobile nav right cluster: push to leading (right) edge */
[dir="rtl"] .ufs-mnav__right { margin-left: 0; margin-right: auto; }

/* btn-icon-left: extra padding moves right */
[dir="rtl"] .btn-icon-left        { padding-left: var(--space-6); padding-right: var(--space-5); }
[dir="rtl"] .btn-sm.btn-icon-left { padding-left: revert; padding-right: var(--space-3); }

/* Form labels: right-align in RTL */
[dir="rtl"] .fi-label,
[dir="rtl"] .signin-forgot { text-align: right; }
