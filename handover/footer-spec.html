Overview

Footer System

The footer is made of two stacked regions — an orange newsletter band and a dark main area — both fully driven by DS tokens. One set of HTML and CSS serves desktop and mobile via a single responsive breakpoint at 768 px.

🟧
Newsletter band
Accessible orange background. Chefs Hand h3 heading, body-sm sub-copy, ghost button + email input. Fully centered column layout.
🔲
Link grid
303 px brand column (logo + tagline) + 3 equal link columns on a --color-brand-dark background. Collapses to 2 columns on mobile.
🌐
i18n ready
All text nodes carry data-i18n / data-i18n-ph attributes. The email input is always dir="ltr". Arabic strings are provided in T_AR.
💡 The footer is placed after #real-page-content and outside any scroll container — it renders as a normal document-flow block. The floating AI bar (.fbe-fixed) is always fixed; padding-bottom: var(--space-25) on .ufs-footer__main ensures content is never obscured by it.

Responsive

Mobile Layout

Below 768 px the footer reorganises into a single-breakpoint responsive layout. The newsletter form stacks vertically and fills full width. The link grid becomes two equal columns. The brand column spans full width with logo and tagline side-by-side, separated from the link columns by a hairline border. The bottom bar inverts — social icons above, copyright below, both centered.

Mobile footer (390 px)
Single @media (max-width: 768px) block handles all layout changes. No JS involved.

i18n & RTL

All visible strings are externalised through data-i18n attributes and populated at runtime via applyTranslations(). The footer supports Arabic out of the box — the grid and flex layouts mirror automatically under dir="rtl".

i18n key reference
Keys used by applyTranslations(). Provide both T_EN and T_AR dictionaries in the page script.

RTL notes
ℹ️ The CSS grid and flexbox layouts mirror automatically when dir="rtl" is set on a parent element. No additional RTL CSS is needed for the footer. The email input must always keep its own dir="ltr" attribute — email addresses are directionally LTR regardless of page language.
⚠️ Do not apply letter-spacing to Arabic text. The --ch-h3-ls: -0.2px on the newsletter heading must be reset to 0 when the page is in Arabic — Arabic is a cursive script and letter-spacing breaks the character joins.
RTL CSS
CSS
/* Reset script letter-spacing for Arabic */
[dir="rtl"] .ufs-footer__nl-heading {
  letter-spacing: 0;
}
